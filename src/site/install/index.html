STUCK?

The common fixes are simple.

I selected a folder and Chrome rejected it

Select the folder containing manifest.json. Do not select the downloaded ZIP, src, registry, or patches.

I cannot see “Load unpacked”

Turn on Developer mode in the upper-right corner of chrome://extensions. On a managed school or work browser, your administrator may disable it.

The repair asks for site access

That is expected once per domain. Chrome shows the exact site before granting it; Patch the Web does not request blanket access during installation.

I clicked “Allow” and the popup disappeared

Open Patch the Web again on the same page. It remembers the approved repair for ten minutes, rechecks the page, and finishes installation automatically.

I installed it but nothing changed

Open the extension on the supported page and choose the verified repair. Patch the Web now reloads once and confirms activation for you. If confirmation fails, choose Retry activation; you do not need to re-import the repair.
