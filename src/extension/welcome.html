A · A REPAIR EXISTS

Install it in one guided flow

  1. 1Visit the website you want to improve.
  2. 2Open Patch the Web from Chrome’s extensions button.
  3. 3Review the exact domain, permissions, selector health, and SHA-256 receipt.
  4. 4Choose Install verified community feature, then allow access to that website.

If Chrome closes the popup: open Patch the Web again. It remembers the approved installation for ten minutes and finishes automatically.

B · NO REPAIR YET

Describe the problem, not the code

  1. 1Stay on the broken page and open Patch the Web.
  2. 2Describe the outcome you want—for example, “show only Karachi programs.”
  3. 3Choose Continue to private request and review the structural brief.
  4. 4Codex authors and tests a constrained repair. Reviewers publish it only after policy and compatibility checks pass.

The brief excludes field values, cookies, storage, page text, and URL queries. You stay in control of what becomes public.

What can a repair do?

Improve layout and accessibility, add safe local progress saving, keyboard controls, validation, search, filtering, comparison, and focused workflows—without arbitrary community JavaScript.
